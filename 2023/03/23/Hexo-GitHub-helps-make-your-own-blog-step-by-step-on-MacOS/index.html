Hexo+GitHub helps make your own blog step by step on MacOS

Posted on 2023-03-23 Edited on 2023-03-24 In Web

Here is a simple tutorial of making a website using Hexo and Github for MacOS users. It’s easy to follow!

To have a website customized for own use seems a common thought of every programmer.

I have tried Gridea before and recently found Hexo a highly free tool to make a website. Strengths of Hexo are like:

  • With an active community
  • Plenty of shared themes and requiring very little coding skills
  • Providing very clear tutorials for beginners to use Hexo
  • Can be deployed to your own server or other code-hosting servers like GitHub.
  • Easily writing contents using Markdown syntax

So here are steps to build a website hosted on GitHub, to be specific, on MacOS.

Build locally

Setup

Please follow Hexo.io setup instruction. The process of specifying a local folder as a Hexo website folder is done in Terminal.

For example, the folder is “/Users/user-name/myblog/“. Firstly, go to the folder.

Then specify the folder as Hexo website folder.

What is generated in the setup process?

We will see the following being generated:

_config.ymlcontains the title of the website, the author and other fundamental information including which theme to use.

package.json

scaffoldscontains the templates of drafts, pages and posts which are contents of the website. Drafts, pages and posts are names of types of layout.

What’s the difference among the layouts?

Layout Type Draft Page Post
Visible Invisible Visible as a button in the navigation bar if it is specified in _config.yml in the theme folder. Visible in homepage, archive page and can be linked with tags and categories.
Path All in ./source/_drafts/. All in ./source/. All in ./source/_posts/.
Filetype draft-name.md Each page correspons the its own folder ./source/page-name/and a web page called index.html in it. post-name.md
Initial status There is no such folder. There is no such folder. The folder _posts is initially set up in ./source/. Usually, there is a markdown file called hello-world.md in it.
Create hexo new draft draft-name hexo new page page-name * hexo new post-name
Publish hexo [layout] <filename>

* Notes:

  • Options are -p or --path(Post path. Customize the path of the post.), -r or --replace(Replace the current post if existed.), -s or --slug(Post slug. Customize the URL of the post.)

  • Tricky points of hexo new page --path ...

hexo new page page-name: create ./source/page-name/index.html

hexo new page --path test/t1 "test" : create ./source/test/t1.mdwith a title of “test”

hexo new page --path test/t1: create ./source/_posts/test/t1.md, which actually creates a markdown file, aka a post.

Design

Web design includes contents, theme, and basic info.

Contents

The contents of the static webpages are written using markdown editors like Typora, Obsidian ,etc.

Theme

_config.yml in ./theme/theme-name/provides optional statements for customization for the style, bars to be shown in the navigation, sidebar, icons and so on. It can be edited in the text/code editor.

Basic Info

_config.yml in ./ (Hexo website folder is the root folder) provides optional statements for customization for the website title, the author, the theme and so on. It can be edited in the text/code editor.

Attachment: hexo commands

or npx hexo instead of hexo.

  • init

  • new

  • Generate: generate or g

    Generate the website folder ./public based on the folder ./source/, the root configuration file ./_config.yml, the theme./theme/and others.

    Option Description
    -d, --deploy Deploy after generation finishes
    -w, --watch Watch file changes
    -b, --bail Raise an error if any unhandled exception is thrown during generation
    -f, --force Force regenerate
    -c, --concurrency Maximum number of files to be generated in parallel. Default is infinity

  • Server: server or s

    Starts a local server. By default, this is at http://localhost:4000/.

    Option Description
    -p, --port Override default port
    -s, --static Only serve static files
    -l, --log Enable logger. Override logger format.

  • deploy: deploy or d

    Option Description
    -g, --generate Generate before deployment

  • render

    Renders files.

    Option Description
    -o, --output Output destination

  • migrate

  • clean

    Cleans the cache file (db.json) and generated files (public).

  • version
